# 🚀 Шпаргалка по настройке ArgoCD в GitHub Codespaces

В этой таблице собраны все необходимые шаги для запуска и получения доступа к интерфейсу ArgoCD.

| Шаг | Действие | Команда / Инструкция | Цель |
| :--- | :--- | :--- | :--- |
| **1** | **Получить пароль** | `kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" \| base64 -d; echo` | Узнать пароль администратора (`admin`) для входа. |
| **2** | **Исправить 502 (Insecure)** | `kubectl patch configmap argocd-cmd-params-cm -n argocd --type merge -p '{"data":{"server.insecure":"true"}}'` | Отключить внутренний HTTPS, чтобы избежать конфликта с прокси GitHub. |
| **3** | **Применить настройки** | `kubectl rollout restart deployment argocd-server -n argocd` | Перезапустить сервер ArgoCD для активации режима *insecure*. |
| **4** | **Создать туннель** | `kubectl port-forward svc/argocd-server -n argocd 8080:80` | Пробросить порт из кластера в воркспейс (не закрывать этот терминал!). |
| **5** | **Открыть доступ** | Вкладка **PORTS** -> Порт 8080 -> Правой кнопкой -> **Visibility: Public** | Сделать порт доступным для внешнего браузера. |
| **6** | **Вход в UI** | Нажать на 🌐 (Open in Browser) во вкладке **PORTS** | Открыть графический интерфейс в новой вкладке. |

---

### 🛠 Решение частых проблем

| Проблема | Причина | Решение |
| :--- | :--- | :--- |
| **Address already in use** | Порт 8080 занят другим процессом | Выполните `killall kubectl` и повторите Шаг 4. |
| **HTTP ERROR 502** | Конфликт сертификатов (SSL) | Убедитесь, что выполнены Шаги 2 и 3, а в Шаге 4 используется порт `:80`. |
| **HTTP ERROR 404** | Неверный формат URL | Удалите `:8080` в самом конце адресной строки браузера. |
| **Панель пуста** | Приложения еще не созданы | Нажмите **+ NEW APP** и укажите ссылку на Git-репозиторий. |
